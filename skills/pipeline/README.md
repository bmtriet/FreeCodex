# FreeCodex Skill Pipeline

The skill pipeline turns repeated work into reusable operating capability.

## Stages

| Stage | Purpose | Input | Output | Done When |
| --- | --- | --- | --- | --- |
| Intake | Capture a public-safe opportunity or repeated workflow. | Lead, PR, audit, or log. | Short opportunity note. | Source is public-safe and bounded. |
| Triage | Decide whether the work is safe and worth doing. | Opportunity note plus policy. | Go/skip decision. | Sensitive or unsafe contexts are removed. |
| Plan | Define the smallest useful action. | Qualified opportunity. | Fix/audit/checklist plan. | Verification path is clear. |
| Execute | Do the work. | Plan and repo context. | PR, report, script, or doc. | Work is complete enough to verify. |
| Review | Check quality and policy fit. | Diff/output and tests. | Pass/fail notes. | Tests or manual checks are recorded. |
| Close | Log outcome and extract learning. | Sent log and result. | Memory note or skill candidate. | Follow-up rule is clear. |

## Promotion Criteria

A workflow becomes a skill when it has:

- Worked at least twice on public-safe tasks.
- A repeatable input/output shape.
- A clear verification method.
- A documented policy boundary.

## Current Candidates

- Public repo safety audit.
- GitHub issue lead scoring with LLMGate.
- Proof-of-work security/readiness PR.
- Outreach log to scoreboard synthesis.

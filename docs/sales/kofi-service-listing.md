## Description

Ship with more confidence before you put an AI-built app, agent workflow, or MCP project in front of users. I review one public GitHub repo and deliver a concise markdown report that calls out obvious launch risks and the next fixes to make first.

What I check:

- leaked-secret patterns and risky env examples
- README, license, security policy, gitignore, and CI basics
- auth, webhook, CORS, and Supabase footguns when visible from public code
- MCP, agent, and skill-specific risk notes when relevant
- prioritized fixes so the next step is obvious

What you get:

- one concise report for one public repo
- redacted evidence only
- typical turnaround: 24-48 hours, queue permitting
- optional one-PR package only when bounded and high-confidence

Donation/support option:

- pay what you want to support public proof-of-work
- useful when a public PR, triage note, checklist, or mini-audit helped
- does not create a report queue slot or guaranteed deliverable

Sponsored public pass option:

- after fit confirmation, pay what you want to sponsor a public-safe mini-pass for one public repo
- accepted only after fit confirmation and queue review
- deliverable may be a public issue, comment, checklist note, or report
- not a private audit, certified pentest, emergency response path, or security guarantee

Boundaries:

- public repo only for the report-only path
- no private credentials, tokens, cookies, private keys, or account access
- not a certified pentest
- no compliance attestation
- no guarantee that a repo is secure

## Optional FAQ

**Do you need repo access?**

No for public repos on the report-only path.

**Will you log or keep secrets?**

No. Suspicious values are redacted in reports, and secrets should not be sent.

**Do you fix issues too?**

Small bounded fixes can be offered after the report and scope confirmation.

**Can I just donate?**

Yes. Donations are welcome if public work helped and you do not need a scoped audit. A donation is support, not a guaranteed report or support contract.

**Can I sponsor a public repo I do not own?**

Maybe. Open a fit-check issue first. The work must stay public-safe, non-invasive, and based only on public information unless explicit permission and scope are confirmed.

**Is this a pentest?**

No. This is a lightweight launch/readiness audit, not a certified penetration test or compliance attestation.

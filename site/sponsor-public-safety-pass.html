How it works

A sponsored pass is smaller than a paid audit and more concrete than a general donation.

1. Sponsor

Use Ko-fi to support a public-safe pass. Any amount is welcome.

2. Submit a public repo

Open the fit-check form with a public GitHub repo URL and launch context only.

3. Fit check

The pass is accepted only if the repo and request are public-safe, bounded, and useful.

4. Public deliverable

The result may be a public issue, comment, checklist note, mini-report, or small PR proposal.

Good fits

Choose a repo where a public note can reduce real launch risk.

Strong candidates handle auth, payments, user data, webhooks, MCP tools, agent workflows, CORS/CSP, Supabase/Firebase rules, dependency audit, or public launch readiness.

This is not a private audit, emergency response, or security guarantee.
